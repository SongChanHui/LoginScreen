# (C# 코딩) 로그인 스크린

## 개요
- C# 프로그래밍 학습
- 1줄 소개: 아이디와 패스워드를 검사하여 일치할 때만 화면을 보여주는 시스템 제작
- 사용한 플랫폼:
  - C#, .NET Windows Forms, Visual Studio, GitHub
- 사용한 컨트롤:
  - Label, TextBox, Button
- 사용한 기술과 구현한 기능:
  - Visual Studio를 이용하여 UI 디자인
  - if~ else~ 조건문을 활용한 상황별 분기 처리
  - MessageBox를 활용한 사용자 알림 및 피드백 제공
  - MessageBox 클래스를 사용하여 로그인 성공 및 실패 상황 알림
  - Color 클래스를 활용하여 힌트 텍스트의 색상을 동적으로 변경
  - 로그인 인증: 아이디와 비밀번호를 비교하여 접근 허용/차단
  - Placeholder 기능을 구현하여 입력창에 안내 메시지 표시



## 실행 화면 (과제1)
- 과제1 코드의 실행 스크린샷

![과제1 실행화면](img/1-1.png)
![과제1 실행화면](img/1-2.png)
![과제1 실행화면](img/1-3.png)

- 과제 내용
  - 메인 화면에 Label(제목), TextBox(아이디와 비밀번호 입력), Button(로그인 실행) 컨트롤을 적절한 위치에 배치합니다.
  - txtID와 txtPW에 각각 Enter 및 Leave 이벤트를 연결하여 실무에서 쓰이는 Placeholder 기능을 상세히 구현합니다.
  - 비밀번호 입력창의 경우 보안을 고려하여 UseSystemPasswordChar 속성이 입력 상태에 따라 유동적으로 변하도록 설정합니다.

- 구현 내용과 기능 설명
  - btnLogin 클릭 시 사용자가 입력한 텍스트를 지정한 아이디와 패스워드를 대조하여 로그인 승인 여부를 판단한다.
  - 아이디와 비밀번호가 모두 일치하면 "로그인 성공!" 메시지 박스를 띄우고, 불일치할 경우 "로그인 실패~" 문구를 출력한다.
  - 입력창이 공백인 상태에서는 안내 힌트가 회색으로 표시되도록 하여 사용자에게 입력 위치를 명확히 인지시킨다.

## 실행 화면 (과제2)
- 과제2 코드의 실행 스크린샷
![과제2 실행화면](img/screenshot-2.png)
- 과제 내용
- Label(표시), TextBox(입력), Button(전송), ListBox(대화창)를 적절히 배치합니다.
- 전송 버튼 클릭 시 TextBox의 텍스트를 ListBox의 항목(Items)으로 추가합니다.
- 추가 직후 TextBox의 내용을 비워(Clear) 다음 입력을 준비합니다.
- 구현 내용과 기능 설명
- 입력창에 메시지 입력하고 전송 버튼을 누르면 메시지가 리스트 박스에 표시된다.
- 계속 반복하면 메시지가 리스트 박스에 한 줄씩 계속 추가된다.
- 추가 내용이 많아지면 리스트 박스에 스크롤바가 자동으로 생기고 스크롤된다.
